### Technical Deep Dive Questions

#### Q1: "What is YOLOv8?"

**Answer:**

> "YOLOv8 stands for 'You Only Look Once' version 8 - a state-of-the-art object detection model. Unlike traditional methods that scan images multiple times, YOLO looks at the entire image once and predicts all objects simultaneously. It uses a 53-layer convolutional neural network with 3.2 million parameters for the nano version. We fine-tuned it on 50 pharmaceutical packaging images to detect holograms, tamper-proof seals, and labels with 89% mAP accuracy."

#### Q2: "How does blockchain work in your system?"

**Answer:**

> "It's blockchain-inspired, not full blockchain. Each supply chain event creates a 'block' containing: drug ID, location, timestamp, and cryptographic hash. Each block's hash incorporates the previous block's hash, creating a chain. If someone tampers with Block 2, its hash changes, breaking the link to Block 3. We can detect tampering instantly using the verify_chain() function which checks all hash links."

#### Q3: "Explain Random Forest classifier."

**Answer:**

> "Random Forest is an ensemble learning method using 100 decision trees. Each tree trains on a random subset of our 10 features (visual from YOLOv8 + behavioral from supply chain). When predicting, all 100 trees vote - majority wins. This prevents overfitting better than a single decision tree. Feature importance shows that travel speed (0.35) and license validity (0.22) are the strongest indicators of counterfeits."

#### Q5: "Why SHA-256 hashing?"

**Answer:**

> "SHA-256 creates a 256-bit (64 character) unique fingerprint of data. Same input always gives same hash, but tiny change completely changes output. It's one-way - can't reverse hash to get original data. Collision-resistant - virtually impossible to find two inputs with same hash. We hash drug details to create tamper-evident ID. If someone changes expiry date, hash verification fails immediately."

**Q: "How do you handle QR photocopy attacks?"**

> "Three layers: (1) Frequency analysis - flag if same QR scanned 15+ times in 1 hour, (2) Geospatial analysis - detect impossible travel speeds, (3) Future: Dynamic QR with time-based OTP that expires every 5 minutes, making photocopies useless."

**Q: "What if database is compromised?"**

> "Blockchain provides audit trail. Even if attacker modifies database, chain verification fails. For production: database encryption at rest, role-based access control, regular backups to immutable storage, separate read-replicas for public queries. Blockchain hashes stored separately can verify database integrity."
